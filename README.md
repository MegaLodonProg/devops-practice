# Dockerfile

---

FROM python:3.11-slim

FROM python:3.11-slim — это инструкция в файле Dockerfile, которая указывает базовый образ для нового Docker-образа. В данном случае python:3.11-slim — это готовый образ, в котором уже установлен Python версии 3.11, а также есть минимальный набор зависимостей. 
Образ содержит не только Python, но и правильную версию ОС (Debian slim), настроенные переменные окружения, pip и т.д. Всё протестировано и оптимизировано.

---

WORKDIR /app

Создаёт папку, если её нет
Переключает текущую директорию на неё для всех следующих COPY, RUN, CMD

---

COPY requirements.txt .

---

RUN pip install --no-cache-dir -r requirements.txt

RUN — инструкция в Dockerfile, которая выполняет команду во время сборки образа. 
--no-cache-dir — флаг, который отключает кэширование загруженных пакетов и метаданных. Это означает, что pip не будет сохранять кэш, и пакеты будут загружаться заново при каждом выполнении команды.

"Никакого venv не надо, потому что это уже изолированная среда"
Абсолютно верно. Каждый контейнер — это уже виртуальное окружение на уровне ОС.

---

VOLUME ["/data"]

WORKDIR — создаёт папку внутри образа (сохраняется в образе)
VOLUME — маркирует папку, которая будет храниться вне контейнера (на хосте), даже если контейнер удалят

WORKDIR просто создаёт папку. При удалении контейнера все данные внутри /app пропадают.
VOLUME говорит Docker: "Эту папку не удаляй, даже если контейнер удалят"

Где физически лежат данные?
Без VOLUME: внутри контейнера (в его слое)
С VOLUME: на хосте (в папке /var/lib/docker/volumes/ или где ты укажешь)

---

EXPOSE 5000

Документирует для других разработчиков: "мой контейнер слушает 5000 порт"

Немного помогает в безопасности (но не блокирует)

Что НЕ делает EXPOSE:
Не открывает порт наружу. Даже с EXPOSE ты не сможешь зайти на localhost:5000, пока не сделаешь -p 8080:5000 или ports: в compose.

Как работает на самом деле:
Внутри контейнера приложение слушает 0.0.0.0:5000

Docker знает, что контейнер слушает 5000 (благодаря EXPOSE или без него — Docker найдёт)

Флаг -p 8080:5000 говорит Docker: "Возьми трафик с порта 8080 на хосте и отправь в порт 5000 контейнера"

---

CMD ["python", "main.py"]  

CMD ["python", "main.py"]      # exec-форма (правильная)
CMD python main.py             # shell-форма (так хуже)
Разница: shell-форма запускает через /bin/sh, что может мешать остановке контейнера по Ctrl+C
